Resso - Документация редактора сайта

Общее описание проекта

Resso - это современный конструктор сайтов, позволяющий пользователям создавать и редактировать свои веб-сайты с помощью удобного визуального редактора. Проект включает в себя множество функциональных блоков и возможностей для настройки.

Основные функции

  • Визуальный редактор с предпросмотром в реальном времени
  • Возможность добавления и редактирования различных типов блоков
  • Сохранение изменений в localStorage
  • Управление видимостью блоков
  • Загрузка и управление изображениями
  • Современный интерфейс с анимациями

Типы блоков

1. Шапка (Header)

  • Название сайта
  • Логотип (с возможностью загрузки)
  • Навигационное меню
  • Фиксированное положение

2. Главный блок (Hero)

  • Заголовок
  • Описание
  • Фоновое изображение
  • Кнопка призыва к действию

3. Услуги

  • Заголовок секции
  • Описание
  • Карточки услуг (3 колонки)
  • Кнопка "Все услуги"
  • Синхронизация с личным кабинетом

4. Портфолио

  • Заголовок секции
  • Сетка работ (3 колонки)
  • Изображения работ
  • Названия работ

5. Контакты

  • Заголовок секции
  • Контактная информация
  • Форма обратной связи
  • Адрес и телефон

6. Пользовательские блоки

  • Текстовый блок (заголовок, описание, изображение)
  • Социальные сети (иконки и ссылки)
  • YouTube видео (встроенный плеер)
  • Галерея (сетка изображений)
  • Отзывы (карточки с рейтингом)
  • Цены (тарифные планы)

Планы по улучшению

  • Добавление возможности перетаскивания блоков (drag and drop)
  • Реализация полноценного предпросмотра сайта
  • Добавление новых типов блоков (блог, FAQ, команда)
  • Улучшение мобильной версии редактора
  • Добавление тем оформления
  • Реализация истории изменений
  • Добавление возможности экспорта/импорта конфигурации
  • Интеграция с системой управления контентом

Технические детали

  • Использование Alpine.js для реактивности
  • Tailwind CSS для стилизации
  • Lucide Icons для иконок
  • Локальное хранение данных в localStorage
  • Адаптивный дизайн
  • Модульная структура кода

Требования к обновлениям

  • Сохранение обратной совместимости с существующими блоками
  • Поддержка всех современных браузеров
  • Оптимизация производительности
  • Улучшение UX/UI
  • Добавление документации для новых функций
  • Тестирование на различных устройствах
